Case Study 1

Website Application

Context

Homepage and key landing pages showcasing the Community Carpool brand system in a web environment.

Brand Elements Used

Header Navigation
Navigation Bar
Logo, menu, CTA button using primary green (#10B981)
Hero Section
Hero Banner
Full-width image + gradient overlay with main headline
Feature Cards
Card Grid
3-column layout with icons, text, and CTA buttons
User Testimonial
Testimonials
Avatar, quote, name, role with matching design system
Footer
Footer Section
Links, social icons, contact info, dark green background
CTA Section
Call-to-Action
Button, form, or signup section with bright green accent

Design System Application

Typography

H1: Montserrat 700, 40px - Page titles
H2: Montserrat 700, 32px - Section headings
Body: Inter 400, 16px - Main content
CTA: Inter 600, 14px - Button text

Color Implementation

Primary: #10B981 for buttons, links, hover states
Accent: #B4E035 for highlights and badges
Text: #1F2937 for body, #6B7280 for secondary
Background: #FFFFFF for cards, #F5F5F5 for sections

Key Takeaways

✓ Success Factors

Consistent use of brand colors, clear typography hierarchy, authentic photography, strong CTA placement, accessible contrast ratios

✗ Avoid

Overuse of bright green, inconsistent button styles, generic stock photos, unclear navigation, poor color contrast

Case Study 2

Mobile App Interface

Context

iOS and Android application implementing the design system within mobile constraints.

Mobile-Specific Adaptations

Element Desktop Mobile Adaptation
Navigation Horizontal menu bar Bottom tab bar (iOS) or hamburger (Android)
Buttons 12px 24px padding 16px 20px padding (larger touch targets)
Typography 16px body text 16px+ (iOS) for tap-friendly interactions
Spacing 24px section gaps 16px gaps (optimize vertical space)
Images Hero 1920×1080px Full-width mobile 375×200px

Key Screens

iPhone Screen
Onboarding Flow
Full-screen images, centered text, primary button
Match Screen
Match Discovery
Card-based layout, avatar, CTA button for connection
Journey Screen
Active Journey
Map view, route details, live status updates

iOS vs Android Considerations

✓ iOS Guidelines

System font (San Francisco), bottom tab bar, large touch targets (44x44pt min), smooth animations, gesture-driven navigation

✓ Android Guidelines

Material Design principles, system back button, 48dp touch targets, hamburger menu, floating action buttons

Brand Consistency Checklist

Use this checklist before approving any marketing material:

Element Checklist Status
Logo ✓ Logo is current version, ✓ Adequate clear space, ✓ Correct colors □ Yes □ No
Colors ✓ Hex/CMYK correct, ✓ Sufficient contrast, ✓ No color drift □ Yes □ No
Typography ✓ Approved fonts only, ✓ Correct sizes & weights, ✓ Hierarchy clear □ Yes □ No
Voice ✓ Tone matches brand, ✓ No typos, ✓ Messaging aligned □ Yes □ No
Imagery ✓ Authentic photos, ✓ Diverse representation, ✓ Minimal filters □ Yes □ No
Layout ✓ Proper spacing, ✓ Design system components, ✓ Responsive ready □ Yes □ No
CTA ✓ Button is primary green, ✓ Clear action text, ✓ Visible placement □ Yes □ No
Accessibility ✓ Alt text on images, ✓ Sufficient contrast, ✓ Readable font size □ Yes □ No
