All APIs & connections

A single reference of every external service Community Carpool connects to, how authentication works, and where each credential lives. Use this as a checklist when onboarding new team members or responding to a security incident.

Anon key vs service role key

Supabase exposes two keys for the project API:

Anon (public) key

  • Safe to include in frontend code — it is designed to be public
  • Subject to Row Level Security (RLS) — cannot access tables directly because no RLS policies are defined (anon is blocked by default)
  • Required in the Authorization: Bearer header when calling edge functions that have verify_jwt: true
  • Stored in config.js which is committed to the public GitHub repo — this is intentional and correct

Service role key

  • Never expose in frontend code — bypasses all RLS, has unrestricted database access
  • Used only inside edge functions via Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')
  • Stored in Supabase Vault and accessible to edge functions at runtime
  • Also stored in the team password manager for Supabase CLI use
🚫
Never commit the service role key to git. If you ever find it in a committed file, treat it as exposed and rotate it immediately. Contact Supabase support for assistance.

Internal connections (within Supabase)

Edge functions connect to the Supabase database internally using the project URL and service role key. These are automatically available inside Supabase edge functions as environment variables:

  • SUPABASE_URL — the project API URL
  • SUPABASE_SERVICE_ROLE_KEY — the service role key
  • SUPABASE_ANON_KEY — the anon key (rarely needed in functions)

Additional secrets (like RESEND_API_KEY) must be set manually via the Supabase CLI or dashboard and are accessed the same way: Deno.env.get('RESEND_API_KEY').

Security checklist

  • Google Maps API key restricted to *.communitycarpool.org/*
  • Resend domain shows "Verified" in the Resend dashboard
  • Service role key is NOT present in any committed file
  • GitHub SSH key or PAT is not shared between team members — each person uses their own credentials
  • All team member logins to external services use unique credentials (not a shared password)

CDN dependencies

Two public CDNs are used passively — no accounts or API keys are required, but an outage would affect the site:

  • esm.sh (https://esm.sh) — imported at the top of every edge function as import { createClient } from 'https://esm.sh/@supabase/supabase-js@2'. If esm.sh is unavailable during a cold start, edge functions will fail to load. This is extremely rare but worth knowing during an incident. Mitigation: Supabase caches imports after the first cold start, so brief outages are unlikely to cascade.
  • Google Fonts (https://fonts.googleapis.com) — loads the Inter typeface on all site pages. If blocked or unavailable, the site falls back to system-ui fonts. No functional impact.
