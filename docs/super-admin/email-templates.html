Email templates

Community Carpool's transactional emails are built as HTML strings directly inside edge function code — there is no separate template engine or external template service. To edit an email, update the HTML string in the relevant function and redeploy.

Template locations

Email typeEdge function fileTrigger
Match notification Supabse Edge Functions/batch-send-emails/index.ts Cron job runs and finds users with unsent matches
Deletion confirmation Supabse Edge Functions/manage-deletion/index.ts User submits a deletion request
Deletion cancellation Supabse Edge Functions/manage-deletion/index.ts User cancels a pending deletion
Deletion completed Supabse Edge Functions/process-deletions/index.ts Cron job processes a scheduled deletion

Email headers (all emails)

HeaderValue
From"Community Carpool" <noreply@communitycarpool.org>
Reply-To[REDACTED] — internal support email
Content-Typetext/html; charset=UTF-8

Editing a template

1

Open the edge function file

Locate the relevant index.ts file in the Supabse Edge Functions/ directory. Find the HTML string — typically defined as a template literal (backtick string) or assembled with string concatenation.

2

Edit the HTML

Make your changes. Keep email HTML simple — avoid complex CSS (inline styles are more reliable across email clients). Test your HTML in an email preview tool like Litmus or Email on Acid if possible.

3

Test with a whitelisted user

Ensure testing_mode = true. Add your email to the whitelist. Deploy to dev and trigger the function. Verify the email looks correct in your inbox.

4

Deploy to production

supabase link --project-ref tbkjealpnoriwdosvmju
supabase functions deploy batch-send-emails

Testing an email end-to-end

  1. Confirm testing_mode = true in config table
  2. Add your email: UPDATE users SET email_whitelist = true WHERE email = 'your@email.com'
  3. Ensure you have an active submission with a new or notified match
  4. In Supabase Dashboard, go to Edge Functions > batch-send-emails > Invoke
  5. Check your inbox and the Resend delivery log
  6. Remove your whitelist flag when done: UPDATE users SET email_whitelist = false WHERE email = 'your@email.com'

Email design principles

  • Use inline CSS — many email clients strip <style> blocks
  • Maximum width: 600px centred — beyond this, many clients clip the email
  • Green (#16a34a) for primary buttons and headings — match the platform branding
  • Keep copy brief and action-oriented — one primary CTA per email
  • Always include a plain text fallback if possible
  • Include an unsubscribe link pointing to communitycarpool.org/unsubscribe.html
💡
After editing a template, check it in at least Gmail and one mobile client before deploying to production. Email rendering varies significantly between clients.
