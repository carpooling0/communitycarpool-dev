The communitycarpool.org domain is managed through a third-party DNS provider. The DNS records serve two purposes: routing web traffic to GitHub Pages, and authenticating outbound email so messages don't land in spam.

Two email providers are configured. Resend authenticates against the root domain (@communitycarpool.org) via its DKIM TXT record. SES authenticates against the send.communitycarpool.org subdomain. Both are ready — the active provider is determined by which secrets are set in Supabase Vault. Currently, RESEND_API_KEY is active; SES is standby.
ℹ️
DNS changes can take up to 48 hours to propagate globally, though most changes are visible within 1–2 hours. Use MXToolbox SuperTool to verify any record type after changes.

GitHub Pages custom domain setup

To verify that the custom domain is configured correctly in GitHub:

  1. Go to the production repo (carpooling0/communitycarpool) on GitHub
  2. Settings > Pages
  3. Under Custom domain, confirm communitycarpool.org is entered
  4. The Enforce HTTPS checkbox should be ticked (requires DNS to propagate first)
  5. GitHub will show a green tick when the domain is verified and HTTPS is active

A CNAME file in each repo root (communitycarpool.org in prod, dev.communitycarpool.org in dev) ties the repo to its domain — do not delete these files.

Email domain verification in Resend

Resend's DKIM TXT record is already set. To verify status:

  1. Go to resend.com/domains
  2. Find communitycarpool.org — status should show Verified
  3. If not verified, the resend._domainkey TXT record may need re-copying from the Resend dashboard — ensure no trailing dot is appended by Porkbun

DMARC progression

The current DMARC policy is p=none with no reporting address — this means failed checks are silently ignored (monitor mode with no visibility). Recommended next steps:

  1. Add a reporting address — update to v=DMARC1; p=none; rua=mailto:carpooling0@gmail.com to start receiving aggregate failure reports
  2. Tighten to p=quarantine — once email delivery is confirmed stable, failed emails go to spam rather than inbox
  3. Move to p=reject — gold standard. Failed emails are rejected outright. Only do this after confirming both Resend and SES (if active) are correctly authenticating.
