Support tickets

Users submit support requests via the public support page. Each submission creates a ticket in the support_tickets table and sends a notification to the internal support email. Admins and Super-Admins can view and respond to tickets.

Ticket types

TypeDescriptionNotes
report_user User reporting another user for inappropriate behaviour Review carefully — consider adding the reported user's email to the blacklist
technical Technical problems (broken link, form error, etc.) Common: expired journey tokens, email not received
general_enquiry General questions about the platform Most common type — usually straightforward to resolve
ℹ️
Deletion requests are not accepted as support tickets. The submit-support-ticket edge function rejects any submission with type deletion with a 400 error and directs users to use the account deletion page (confirm-deletion.html).

Viewing tickets via SQL

Until the admin panel Tickets panel is live, query tickets directly:

-- View all open tickets, newest first
SELECT ticket_id, email, type, message, created_at, status
FROM support_tickets
WHERE status = 'open'
ORDER BY created_at DESC;

-- View tickets from a specific email
SELECT * FROM support_tickets WHERE email = 'user@example.com' ORDER BY created_at DESC;

Handling a ticket

1

Review the ticket

Read the message and type. For report_user tickets, look up the reported user in the users table to assess the situation.

2

Add admin notes

Record your actions and findings:

UPDATE support_tickets
SET admin_notes = '[2026-03-15 09:00 GMT+4] Admin Name: Reviewed. Issue confirmed. Sent replacement journey link to user.'
WHERE ticket_id = 42;

Admin notes should be append-only — prefix each entry with timestamp and your name.

3

Take action

Depending on ticket type: generate a new journey token for the user, investigate a technical issue, or add a reported email to the blacklist.

4

Close the ticket

UPDATE support_tickets
SET status = 'closed', closed_at = now()
WHERE ticket_id = 42;

Notifying the submitter

There is no automated reply mechanism yet. To notify the user of your response:

  • Reply directly from the internal support email ([REDACTED]) to the ticket submitter's email
  • Or use Resend to send a one-off email (not yet built into the admin panel)

Internal support email

When a new ticket is submitted, a notification is sent to the support_notify_email config value: [REDACTED]. This is an internal address that forwards to the team. Check this inbox regularly.

Escalation to Super-Admin

The Support role can read and update tickets but cannot delete users or access full user data. For tickets requiring user deletion or investigation of user account details, escalate to an Admin or Super-Admin.

report_user tickets that appear credible should be escalated — only an Admin or Super-Admin can add to the blacklist or process a deletion.
