GitHub & deployment

Community Carpool uses two GitHub repositories — one for production and one for development — both hosted on GitHub Pages. Deployment is manual: push to the correct remote to update the live site. Edge functions are deployed separately via the Supabase CLI.

Repositories

RepositoryPurposeGitHub Pages URL
carpooling0/communitycarpool Production — live site, real users communitycarpool.org (custom domain)
carpooling0/communitycarpool-dev Development — safe to experiment carpooling0.github.io/communitycarpool-dev

GitHub credentials (username/PAT or SSH key): [REDACTED] — see password manager.

Local git remotes

In your local clone, verify the remotes are configured correctly:

git remote -v

Expected output:

dev-repo   git@github.com:carpooling0/communitycarpool-dev.git (fetch)
dev-repo   git@github.com:carpooling0/communitycarpool-dev.git (push)
origin     git@github.com:carpooling0/communitycarpool.git (fetch)
origin     git@github.com:carpooling0/communitycarpool.git (push)
⚠️
Always verify remotes before pushing. Pushing to origin deploys to the live production site immediately. There is no staging buffer or CI gate.

Deploying to development

Push your local dev branch to the dev repo's main branch:

git push dev-repo dev:main

GitHub Pages auto-deploys within 1–2 minutes. Check the dev site to verify changes.

Deploying to production

🚫
This is LIVE immediately. Only push to production when changes have been tested on dev and you are confident they are correct.
# Ensure you are on the main branch and it is up to date
git checkout main
git merge dev   # or cherry-pick specific commits

# Push to production
git push origin main

GitHub Pages deploys the main branch of the production repo. The custom domain communitycarpool.org is configured in repo Settings > Pages.

What GitHub Pages serves

GitHub Pages serves static HTML, CSS, and JS files directly from the repository root. There is no build step. All page logic is vanilla JS in the HTML files, and all dynamic data comes from Supabase edge functions via fetch() calls.

Edge functions — separate deployment

Edge functions are not deployed via git push. They live in the Supabse Edge Functions/ directory (note the typo in the folder name — do not rename it) but are deployed using the Supabase CLI. See Deploying edge functions for the full process.

What NOT to commit

  • .env files — never commit any file containing API keys or secrets
  • config.dev.js — this contains dev environment config, check before committing
  • Service role keys — if these appear in any file, remove them immediately
  • Personal data — no database exports, user lists, or PII should ever be committed
💡
Add a .gitignore that excludes *.env, .env*, config.dev.js, and *.zip to prevent accidental credential commits.

SSL / HTTPS certificates

HTTPS is provided automatically by GitHub Pages via Let's Encrypt. There is nothing to set up, pay for, or renew manually.

When a custom domain is configured in repo Settings > Pages and the DNS CNAME is pointing to carpooling0.github.io, GitHub automatically:

  1. Requests a TLS certificate from Let's Encrypt for that domain
  2. Renews it every 90 days in the background
  3. Enforces HTTPS (HTTP requests are redirected to HTTPS)

This applies to both domains:

DomainCertificate issued byCostRenewal
communitycarpool.orgLet's Encrypt (via GitHub Pages)FreeAutomatic — no action needed
dev.communitycarpool.orgLet's Encrypt (via GitHub Pages)FreeAutomatic — no action needed

To verify: go to carpooling0/communitycarpool → Settings → Pages. You will see a green "✓ Your site is published at https://communitycarpool.org" and an "Enforce HTTPS" checkbox (should be ticked).

💡
If the certificate ever shows as invalid or expired, it usually means the DNS CNAME was temporarily broken. Fix the DNS record and GitHub Pages will re-provision the certificate within a few minutes.

Checking GitHub Pages deploy status

Go to the repository on GitHub > Actions tab. GitHub Pages deployments appear as workflow runs. A green tick means the deploy succeeded. If it fails, click the run to see the error.
