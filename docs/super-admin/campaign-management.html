Campaign management

Community Carpool uses referral links and QR codes to onboard partner organisations. Campaign tracking is currently manual — analytics are pulled via SQL queries. This page covers how to generate links, track performance, and maintain brand consistency.

Link types

PurposeURL formatEffect
Org pre-fill communitycarpool.org/?org=ORGCODE Pre-fills the organisation field on the submission form — user doesn't need to select their org manually
Referral tracking communitycarpool.org/?ref=CODE The ref parameter can be read by analytics — allows tracking where submissions came from
Combined communitycarpool.org/?org=ORGCODE&ref=SOURCE Pre-fills org and tracks source simultaneously

Generating QR codes

Use any QR code generator to create a code from a referral URL. The QR code should point to the full URL including query parameters:

Example: https://communitycarpool.org/?org=ACME&ref=poster-q1-2026

Recommended tools: qr-code-generator.com, qrcodemonkey.com, or any offline tool. Download as SVG for print quality.

Include the QR code on:

  • Printed posters (A4/A3 for noticeboards)
  • Internal email campaigns to employees
  • Intranet pages or Slack announcements

Tracking campaign performance

Since there is no dedicated analytics dashboard, use SQL queries to measure campaign results:

-- Total submissions by org_code (all time)
SELECT org_code, COUNT(*) as total
FROM submissions
GROUP BY org_code
ORDER BY total DESC;

-- Submissions by org_code in the last 30 days
SELECT org_code, COUNT(*) as recent_submissions
FROM submissions
WHERE created_at > now() - INTERVAL '30 days'
GROUP BY org_code
ORDER BY recent_submissions DESC;

-- New users in the last 7 days (total and by org)
SELECT
  org_code,
  COUNT(*) as new_users,
  MIN(created_at) as first_signup,
  MAX(created_at) as last_signup
FROM users
WHERE created_at > now() - INTERVAL '7 days'
GROUP BY org_code
ORDER BY new_users DESC;

Brand guidelines

Maintain brand consistency across all campaign materials:

  • Primary colour: green #16a34a (or dark variant #15803d)
  • Font: Inter (available on Google Fonts)
  • Logo: use logo.png or logo_with_slogan.png from the repo root
  • QR logo code: logo-qr.png is a version optimised for QR code overlays
  • Tone: friendly, community-focused, sustainability-positive — not corporate

Planned: UTM parameter support

Future development will add UTM parameter tracking (utm_source, utm_medium, utm_campaign) so that email campaigns, social posts, and poster campaigns can be distinguished in analytics without separate SQL queries. This requires frontend code to read and store UTM values with each submission.

Planned: Campaign management UI

A dedicated Campaign Management section in the admin panel is planned for future development. It would allow generating tracked links, viewing conversion metrics, and managing active campaigns without requiring SQL access.
