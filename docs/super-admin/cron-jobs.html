⚡ Edge Functions & Automation

Cron jobs & scheduled tasks

Two edge functions run on a schedule via Supabase's built-in cron scheduler (powered by pg_cron): one sends match notification emails, the other processes pending account deletions. All scheduled times are in UTC.

Scheduled functions

Function Schedule (UTC) GMT+4 equivalent Purpose
process-deletions 0 22 * * *
(daily at 22:00 UTC)		 02:00 GMT+4 daily Processes scheduled account deletions where the grace period has elapsed. Deletes user data and writes to deletion_log.
batch-send-emails Check Supabase dashboard for current schedule — Sends match notification emails to users with status new or notified that haven't been emailed yet. Respects testing_mode.

Cron expression reference

Cron expressions follow the format: minute hour day-of-month month day-of-week

# Run daily at 22:00 UTC
0 22 * * *

# Run every hour
0 * * * *

# Run every day at 08:00 UTC (12:00 GMT+4)
0 8 * * *

# Run every Monday at 06:00 UTC (10:00 GMT+4)
0 6 * * 1

How cron jobs authenticate

Both scheduled functions have verify_jwt: true. When Supabase's scheduler invokes an edge function internally, it automatically provides a valid service-level JWT — no manual configuration is needed. This means the cron trigger is privileged and the functions can access the database with the service role.

If you were to call these functions manually from the browser with only the anon key, you would get a 401 — which is the intended security behaviour. Only the scheduler (or a client with the service role key) can invoke them successfully.

Viewing and managing cron jobs

Via Supabase dashboard

  1. Go to the Supabase dashboard for the production project
  2. Navigate to Edge Functions in the left sidebar
  3. Click on a function (e.g. batch-send-emails)
  4. Go to the Schedules tab to see and edit the cron schedule

Alternatively, cron jobs can be viewed under Database > pg_cron if the pg_cron extension is visible.

Via SQL (pg_cron)

-- List all scheduled jobs
SELECT jobid, jobname, schedule, command FROM cron.job;

-- Delete a job (replace jobid with actual ID)
SELECT cron.unschedule(jobid) FROM cron.job WHERE jobname = 'batch-send-emails';

Manually triggering a function

You can trigger a scheduled function manually from the Supabase dashboard for testing:

  1. Go to Edge Functions in the Supabase dashboard
  2. Click on the function name
  3. Click Invoke — this sends an empty POST request with the service role key
  4. View the response and logs in the function's Logs tab
⚠️
Manually triggering batch-send-emails on production will send real emails to any whitelisted users (or all users if testing_mode is disabled). Confirm the testing_mode setting before triggering manually.
⚠️
Manually triggering process-deletions on production will permanently delete user accounts that have passed their grace period. This cannot be undone.

Checking if a cron job ran successfully

  1. Go to Logs > Edge Functions in the Supabase dashboard
  2. Filter by the function name
  3. Check for entries at the expected schedule time
  4. Look for HTTP 200 responses and expected console.log output (e.g. emails sent count, deletions processed count)
  5. Error entries (5xx) indicate the function failed — investigate the error message
