Config table

The config table is a simple key-value store that controls platform behaviour at runtime. Edge functions read from this table on each request — changes take effect immediately without redeploying code.

How to read the config table

-- View all config values
SELECT key, value FROM config ORDER BY key;

How to update a config value

-- Update a single value
UPDATE config SET value = 'false' WHERE key = 'testing_mode';

-- Update multiple values
UPDATE config SET value = '60' WHERE key = 'polling_interval_seconds';
UPDATE config SET value = '1.2' WHERE key = 'required_terms_version';
⚠️
Changes are live immediately. Updating testing_mode to false means the next cron run of batch-send-emails will send emails to all users with pending notifications. Think carefully before changing operational flags in production.

Config keys reference

Key Current value Type Description
testing_mode true Boolean (as string) When true, batch-send-emails only sends to users with email_whitelist = true. Set to false to go live with emails. See Testing mode.
match_notification_enabled true Boolean (as string) Master switch for match email notifications. When false, no emails are sent regardless of testing_mode. Use this to pause email sending without disabling the cron job.
data_retention_days 30 Integer (as string) Grace period in days after a deletion request before process-deletions permanently removes user data. The 30-day period gives users time to cancel accidental deletion requests.
match_token_expiry_days 120 Integer (as string) How many days a journey token (used in the matches page URL) remains valid. After expiry, the user sees a "link expired" message and must contact support to get a new link.
support_notify_email [REDACTED] Email string Internal email address that receives a notification when a new support ticket is submitted. This is not shown to users.
required_terms_version [Current version — check DB] Version string (e.g. "1.1") The T&C version that all users must have accepted. If a user's terms_accepted_version differs from this value, submit-journey returns a 409 and the frontend shows the T&C modal. See T&C process.
terms_last_updated [Date — check DB] Date string (e.g. "15 March 2026") Human-readable date shown in the T&C modal so users know when the terms were last changed.
polling_interval_seconds 10 Integer (as string) How often matches.html polls the get-matches-page function for updates. 10 seconds is aggressive for development/testing — change to 300 (5 minutes) or higher in production to reduce API call volume.

Adding a new config key

-- Insert a new config key
INSERT INTO config (key, value) VALUES ('new_feature_enabled', 'false')
ON CONFLICT (key) DO UPDATE SET value = EXCLUDED.value;

After adding, update the edge function code to read the new key using the same pattern as existing keys.

Config reading pattern in edge functions

Edge functions read config values like this:

const { data: configRows } = await supabase
  .from('config')
  .select('key, value');

const config = Object.fromEntries(configRows.map(r => [r.key, r.value]));

const testingMode = config['testing_mode'] === 'true';
const retentionDays = parseInt(config['data_retention_days'] ?? '30', 10);

All values are stored as text and must be cast appropriately (boolean comparison against the string 'true', integer parsing, etc.).
