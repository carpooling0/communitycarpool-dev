Connections reference

Every integration listed with direction, protocol, and what breaks if it goes down.

From To How What it does If it fails
User browser GitHub Pages HTTPS (via Porkbun DNS → A records) Loads all site pages (index.html, matches.html, etc.) Site unreachable. Check Porkbun DNS + GitHub Pages status.
User browser (index.html) Google Maps Places API HTTPS (direct from browser, API key in URL) Address autocomplete in "From" and "To" fields on the submission form Autocomplete stops working. User must type the full address manually. Form still submits.
All site pages Google Fonts CDN HTTPS (passive, browser loads on page render) Loads the Inter typeface used across all pages Site falls back to system-ui fonts. No functional impact.
Frontend pages Supabase Edge Functions HTTPS POST/GET with Authorization: Bearer {anon_key} All user actions: submit journey, view matches, express interest, submit ticket, update email prefs All user-facing actions stop working. Check Supabase Edge Function logs.
admin.html admin-auth + admin-api edge functions HTTPS POST with session token (no JWT required — custom auth) All admin panel operations: metrics, tickets, deletions, user lookup, blacklist, team management Admin panel login fails or actions error. Check Supabase Edge Function logs for admin-api.
Edge Functions PostgreSQL (Supabase DB) Internal Supabase client (service role key, no network hop) All database reads and writes: users, submissions, matches, config, events, etc. All edge functions error. DB may be paused (free tier) or overloaded. Check Supabase DB health.
submit-journey, batch-send-emails, update-match-status Resend API HTTPS POST to api.resend.com/emails with Bearer token Sends all transactional emails: match notifications, mutual match contact reveals, deletion confirmations Emails not sent. Matches are found and stored but users aren't notified. Switch to SES if Resend is down.
submit-journey, batch-send-emails, update-match-status AWS SES HTTPS POST to SES endpoint (AWS4-HMAC-SHA256 signed) Fallback email provider — activates automatically if RESEND_API_KEY is unset N/A if Resend is active. If SES is primary and fails: emails stop, check AWS SES console for sandbox/quota limits.
submit-journey Mapbox Directions API HTTPS GET to api.mapbox.com/directions/v5/mapbox/driving/… Calculates driving distance between origin and destination for each new journey submission Falls back to haversine (straight-line) distance. Match quality is minimally affected. Check MAPBOX_TOKEN in Vault.
Resend resend-webhook edge function HTTPS POST from Resend to our webhook URL (registered in Resend dashboard) Receives email delivery events (delivered, bounced, opened) and writes to email_events table Delivery analytics stop updating. No user impact — emails are still sent.
Supabase Cron Edge Functions Internal Supabase scheduled task (pg_cron) Triggers batch-send-emails, expire-journeys, process-deletions on schedule Scheduled jobs stop running. Reminders not sent, expired journeys not archived, pending deletions not processed. Check Supabase Cron logs.
Edge Functions (cold start) esm.sh CDN HTTPS import at Deno runtime initialisation Fetches @supabase/supabase-js package on first cold start Functions fail to start if esm.sh is down during a cold start. Cached after first run — brief outages unlikely to cascade.
Porkbun DNS GitHub Pages DNS A records (185.199.x.153) + CNAME for www/dev Routes communitycarpool.org → GitHub Pages servers Site unreachable if DNS propagation lapses. Check A records in Porkbun DNS panel.
GitHub Pages Let's Encrypt ACME protocol (automatic, managed by GitHub) SSL/TLS certificate for communitycarpool.org — auto-renewed by GitHub HTTPS stops working (browser shows security warning). Log in to GitHub repo Settings → Pages and trigger certificate renewal.
