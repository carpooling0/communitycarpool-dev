Email — Resend & SES

Community Carpool sends transactional emails via Resend. AWS SES is documented as a future alternative. This page covers how email sending works, where to manage it, and how to switch providers if needed.

How Resend is called

Edge functions call the Resend API directly over HTTPS:

const res = await fetch('https://api.resend.com/emails', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${Deno.env.get('RESEND_API_KEY')}`,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    from: 'Community Carpool <noreply@communitycarpool.org>',
    to: [userEmail],
    subject: 'Your carpool matches are ready',
    html: emailHtml,
  }),
});

Viewing email delivery logs

Go to resend.com/emails — this shows all sent emails with delivery status (delivered, bounced, clicked). If a user reports not receiving an email, check here first before investigating the edge function logs.

DNS records required

For emails to be delivered reliably and not land in spam, the following DNS records must be in place for communitycarpool.org:

  • SPF — authorises Resend to send on behalf of your domain
  • DKIM — cryptographic signature for email authentication
  • DMARC — policy for handling failed SPF/DKIM checks

See DNS records for the exact values. Resend's dashboard shows domain verification status — check that the domain shows as "Verified".

Resend webhook (future)

Resend can call a webhook URL when emails are opened, bounced, or clicked. A resend-webhook edge function exists in the codebase for this purpose but is not yet active. When enabled, it would allow tracking email engagement and handling bounces automatically (e.g. marking a user's email as invalid).

Switching to AWS SES (future)

If Resend's pricing or limits become a constraint, the platform can switch to AWS Simple Email Service (SES). The steps to switch are:

1

Set up SES in AWS Console

Create an AWS account, verify the communitycarpool.org domain in SES, and move out of the SES sandbox (request production access).

2

Create an IAM user with SES permissions

Create an IAM policy with ses:SendEmail permission and generate access keys.

3

Set Supabase secrets

supabase secrets set AWS_REGION=us-east-1
supabase secrets set AWS_ACCESS_KEY_ID=[KEY]
supabase secrets set AWS_SECRET_ACCESS_KEY=[SECRET]
supabase secrets set SES_FROM_EMAIL=noreply@communitycarpool.org
4

Update edge functions

Replace the Resend fetch() calls with the AWS SDK SESClient. Important: initialise the client lazily inside the request handler (not at module level) to avoid cold-start 502 errors — a bug that was previously encountered and fixed.

5

Update DNS records

Add SES-specific DKIM records. SPF must include SES: include:amazonses.com.
