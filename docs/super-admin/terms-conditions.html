T&C process

Community Carpool tracks which version of the Terms & Conditions each user has accepted. When the T&C are updated, the system requires all users to re-accept before they can submit a new journey.

How it works

The system compares two values when a user submits a journey:

  • config.required_terms_version — the current required version (e.g. "1.2")
  • users.terms_accepted_version — the version the user last accepted

If they differ, submit-journey returns a 409 Conflict response with:

{
  "error": "terms_required",
  "requiredVersion": "1.2",
  "termsLastUpdated": "15 March 2026"
}

The frontend intercepts this response and shows a modal with the T&C. When the user clicks Accept, the frontend resends the journey submission with acceptTerms: true in the request body. submit-journey then updates users.terms_accepted_version to the required version and processes the submission normally.

T&C document

The terms document lives at terms.html in the site root. It is served as a static HTML page at communitycarpool.org/terms.html. Edit this file in the repository and push to update the live terms.

Config keys involved

Config keyExample valuePurpose
required_terms_version"1.2"The version number all users must have accepted. Incrementing this triggers re-acceptance for all existing users.
terms_last_updated"15 March 2026"Human-readable date shown in the T&C modal. Update this when you update the version.

DB fields involved

FieldTablePurpose
terms_accepted_versionusersThe T&C version the user most recently accepted. Updated on each submission where user accepts.
terms_accepted_atusersTimestamp of most recent acceptance. Useful for compliance records.
terms_versionsubmissionsThe T&C version that was in effect when this submission was created.

Updating the T&C — step by step

1

Edit terms.html

Make the required changes to terms.html in the repository. Keep a clear section at the top showing the version number and effective date.

2

Update config table

UPDATE config SET value = '1.2' WHERE key = 'required_terms_version';
UPDATE config SET value = '15 March 2026' WHERE key = 'terms_last_updated';

This takes effect immediately — the next user to submit a journey will see the T&C modal.

3

Push terms.html to production

git add terms.html
git commit -m "Update T&C to v1.2 — [brief description of change]"
git push origin main
4

Verify

Submit a test journey from an existing test user account. Confirm the T&C modal appears. Accept and confirm the submission proceeds normally.

ℹ️
Users are only prompted to re-accept when they next try to submit a journey. Users who don't submit new journeys will not see the modal — their existing submissions and matches remain active regardless.

Checking T&C acceptance status

-- How many users have accepted the current version?
SELECT
  COUNT(CASE WHEN terms_accepted_version = (SELECT value FROM config WHERE key = 'required_terms_version') THEN 1 END) AS accepted,
  COUNT(CASE WHEN terms_accepted_version != (SELECT value FROM config WHERE key = 'required_terms_version') OR terms_accepted_version IS NULL THEN 1 END) AS not_yet_accepted,
  COUNT(*) AS total_users
FROM users;
