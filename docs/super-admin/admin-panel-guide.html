Admin Panel Guide

A complete reference for every panel in the Community Carpool admin panel — from logging in and reading the dashboard to handling deletions, managing the team, and exporting audit logs.

📘
Admin Panel Guide exists as a standalone doc set. Admin and Support team members have their own focused guide at docs/admin/ — a subset covering only the panels they can access, without infrastructure or secrets content. This Super-Admin page covers all panels including Team, Audit Log, and system-level context that Admins and Support roles do not see.

Logging in

Authentication

The admin panel is accessible at:

https://communitycarpool.org/admin.html

Enter your email address and password on the login screen. Both are [REDACTED] — contact a super_admin to obtain or reset credentials.

Session expiry: Sessions remain active for as long as the browser tab is open. Closing the tab or navigating away does not immediately log you out, but the session token expires after a period of inactivity. If you see a "Session expired" or authentication error, simply log in again. Always log out manually (profile dropdown → Log out) when using a shared or public device.

The admin panel uses its own credential system (the admin_users table), separate from the Supabase auth system used by end users. Your admin credentials do not give you access to the Supabase dashboard and vice versa.

Dashboard panel

Dashboard

The Dashboard is the first panel you see after logging in. It gives a live snapshot of platform health:

MetricWhat it shows
Total usersAll registered users (distinct emails that have submitted a journey at least once).
Active submissionsJourney submissions with journey_status = 'active' — users currently seeking a carpool partner.
Total matchesAll match pairs created by the matching engine, regardless of status.
Open support ticketsSupport tickets with status open that have not yet been resolved.

Growth chart: Below the metric cards is a line chart showing new submissions over time (grouped by week or month). Use this to spot seasonal trends, measure the impact of a campaign launch, or identify unexpected drop-offs in new sign-ups.

Dashboard metrics are fetched live from the database each time you open the panel. There is no caching — what you see reflects the current state of the production database.

Tickets panel

Tickets

The Tickets panel is where support requests submitted via the platform's support form are managed.

Viewing and filtering tickets

Tickets are listed in reverse chronological order (newest first). Use the status filter tabs to switch between:

  • open — tickets awaiting a response or action
  • closed — tickets that have been resolved
  • All — shows every ticket regardless of status

Each ticket row shows the submitter's email, ticket type (e.g. technical, report_user), submission date, and current status.

Expanding a ticket

Click any ticket row to expand it and see the full message body, any internal notes, and the reply history.

Adding an internal note

Internal notes are visible only to admin team members — they are never sent to the user. Use them to record investigation steps, assign responsibility, or flag for follow-up. Type in the "Internal note" field and click "Save note".

Sending a reply to the user

Type your response in the "Reply to user" field. Tick "Notify user" if you want the reply to be emailed to the user's address. Click "Send reply". The reply is saved against the ticket in the database. If "Notify user" is ticked, an email is dispatched via the email provider (Resend).

What "Notify user" does

When ticked, the platform sends a transactional email to the user's address containing your reply text. If unticked, the reply is saved internally but the user receives no email notification. Untick it when making administrative notes that are not ready to share with the user.

Closing a ticket

Once the issue is resolved, click "Close ticket". The status changes to closed and the ticket moves to the Closed tab. Closed tickets can be re-opened if the user follows up.

Deletions panel

Deletions

Users can request deletion of their account and data via the platform's unsubscribe flow. These requests are queued and processed either automatically by the nightly cron job or manually by an admin from this panel.

Viewing pending deletion requests

The panel lists all users with a pending deletion request (deletion_requested_at IS NOT NULL and deletion_processed_at IS NULL). For each request you can see:

  • The user's email address
  • The date and time they requested deletion
  • A countdown timer showing how long until the request is automatically processed

What the timer means

The platform enforces a cooling-off window (configured by data_retention_days in the config table — currently 30 days). Users have this window to cancel their request before data is permanently deleted. The timer in the panel shows the remaining cooling-off time. Once it reaches zero, the nightly cron job processes the deletion automatically.

Cancelling a deletion (admin-initiated)

An admin can cancel a pending deletion request on behalf of a user — for example, if the user contacts support saying they changed their mind and cannot find the cancellation link. To cancel:

  1. Locate the user in the Deletions panel.
  2. Click "Cancel deletion".
  3. Confirm the action in the prompt.

Cancelling a deletion clears deletion_requested_at and deletion_token on the user's record. Their journey submission is restored to active status. The user receives a confirmation email that their deletion has been cancelled and their account is active again.

Processing (confirming) a deletion manually

If you need to process a deletion immediately — for example, in response to a formal data subject erasure request — rather than waiting for the cron:

  1. Locate the user in the Deletions panel.
  2. Click "Process now".
  3. You will be prompted to enter your 6-digit deletion PIN. This is a separate PIN from your login password, set when your admin account was created (or changed in your profile settings). It acts as a second confirmation to prevent accidental irreversible deletions.
  4. Enter the PIN and confirm.
Deletion is irreversible. Once processed, the user's personal data is permanently removed or anonymised. This includes their name, email, submission details, and match history. This action cannot be undone. Always double-check you have the correct user before proceeding.

What gets deleted

Processing a deletion removes or anonymises:

  • The user's row in the users table (name, email, phone, org affiliation)
  • Their active journey submission(s)
  • Their match records (both sides of any match they were part of)
  • Their email preferences
  • Any support tickets they submitted

Users panel

Users

The Users panel lets you look up any user by email address and inspect or manage their account.

Searching by email

Type the full or partial email address in the search field and press Enter. Matching users are listed below. Click a result to expand the user's full profile.

What is shown

The expanded user profile shows:

  • Profile: Name, email, registration date, org affiliation (if any), and current blacklist status.
  • Submissions: All journey submissions associated with this user — origin, destination, days/times, and current status.
  • Matches: All matches the user is part of, with match strength, the other party's details, and the current match status (new, interest_expressed, contact_revealed, expired).
  • Blacklist status: Whether the user is currently blacklisted and, if so, the reason and the admin who applied it.

Blacklisting a user

Blacklisting prevents a user from receiving new matches and from appearing as a potential match for other users. To blacklist:

  1. Find the user via the search field.
  2. Click "Blacklist user".
  3. Enter the reason (this is recorded in the audit log and visible to all admins).
  4. Confirm.

The user's is_blacklisted flag is set to true. They are not notified. Their existing active submission remains in the database but is skipped during match-finding.

Removing a blacklist

If a blacklist was applied in error or the situation has been resolved:

  1. Find the user in the Users panel.
  2. Click "Remove blacklist".
  3. Confirm.

The is_blacklisted flag is cleared. The user will be included in future match-finding runs again. The unblacklist action is recorded in the audit log.

Campaign Links panel

Campaign Links

The Campaign Links panel generates trackable referral links and QR codes for distribution in marketing campaigns or partner onboarding packs.

Two modes

ModeWhat it generatesWhen to use
Individual link A single URL with a unique identifier, designed to be shared with one person or a small group. Useful for tracking which specific individual or channel drove a sign-up. Sending to a specific person, influencer, or community leader whose referrals you want to attribute individually.
Campaign link A URL with UTM parameters (utm_source, utm_medium, utm_campaign) for bulk distribution. All click-throughs from this link are grouped under the same campaign in analytics. Email blasts, social media posts, printed materials, or partner org referral links.

UTM parameters

When generating a Campaign link, you fill in three fields that become UTM parameters on the URL:

FieldURL parameterExample valueWhat it tracks
Sourceutm_sourceemail, whatsapp, posterThe channel or platform the link was placed on
Mediumutm_mediuminternal, social, printThe marketing medium
Campaignutm_campaignacme-launch-mar26The specific campaign name for grouping in analytics

Generating a QR code

After filling in the link details, click "Generate QR code". A QR image is displayed on screen. Click "Download QR" to save it as a PNG file suitable for printing on posters, flyers, or email graphics.

Copying the link

Click "Copy link" next to the generated URL to copy it to your clipboard. Paste it into emails, WhatsApp messages, or any other distribution channel.

Audit Log panel

Audit Log

The Audit Log records every significant admin action taken within the admin panel. It is the primary accountability trail for the platform.

What is recorded

Every entry includes:

  • Timestamp: The exact date and time the action was taken (UTC).
  • Admin: The name and email of the admin team member who performed the action.
  • Action: A description of what was done (e.g. "Blacklisted user", "Cancelled deletion", "Closed ticket", "Added team member").
  • Entity email: The email address of the user or entity that was affected by the action.
  • Notes: Any additional context recorded at the time (e.g. the blacklist reason).

Filtering the log

Use the filter controls to narrow the log by:

  • Date range: Show only entries within a specific period.
  • Admin: Show only actions performed by a specific team member.
  • Action type: Filter to a specific category of action (e.g. blacklist, deletion, ticket).

Exporting to CSV

Click "Export CSV" to download the current filtered view as a CSV file. Use this for compliance reporting, incident investigations, or sharing with a super_admin for review.

Team panel

Team

The Team panel is where super_admins manage who has access to the admin panel and at what permission level.

The three roles

RoleWhat they can access
support Tickets panel only. Can view, reply to, and close tickets. Cannot access Users, Deletions, Audit Log, Team, or Campaign Links. Suitable for frontline support staff.
admin Everything support can do, plus: Users panel (search, blacklist, unblacklist), Deletions panel (view and cancel deletions — but cannot process/confirm a deletion without a deletion PIN), Campaign Links. Cannot manage team members or view the Audit Log.
super_admin Full access to all panels including Team management, Audit Log, and the ability to process (confirm) deletions. Can add, edit, and deactivate any team member. Should be restricted to two or three trusted individuals.

Adding a new team member

  1. Go to Team panel and click "Add team member".
  2. Fill in the following fields:
FieldNotes
NameThe team member's full name. Appears in the audit log against their actions.
EmailTheir email address. Used as their login username.
RoleSelect support, admin, or super_admin.
PasswordSet an initial password. The team member should change it on first login (profile dropdown → Change password).
Deletion PINA 6-digit PIN required to confirm irreversible deletion actions. Required for all roles, even if the role cannot perform deletions — it must be set for future role upgrades. Communicate it to the team member separately from the password.
Access expiry dateOptional. If set, the account is automatically deactivated on this date. Useful for contractors or temporary cover. Leave blank for permanent access.
  1. Click "Save". The new team member can now log in immediately.
Never share credentials over unencrypted channels. Send the email address and a temporary password via separate secure messages. Instruct the new team member to change their password on first login.

Editing an existing team member

Click the edit icon next to any team member in the list to modify their name, role, or access expiry date. You can also reset their password from here if they are locked out.

Deactivating access

To revoke access for a team member who has left or no longer needs access:

  1. Click the edit icon next to their row.
  2. Toggle "Active" to off (or set an expiry date in the past).
  3. Save.

Deactivated accounts cannot log in. Their name still appears in the audit log against historical actions they took. Do not delete team member records — deactivate them so the audit trail is preserved.

Changing your password and deletion PIN

Profile

Your login password and deletion PIN can both be changed from your profile dropdown:

  1. Click your name or avatar in the top-right corner of the admin panel.
  2. Select "Change password" or "Change deletion PIN" from the dropdown menu.
  3. For a password change: enter your current password, then your new password twice, and confirm.
  4. For a PIN change: enter your current PIN, then your new 6-digit PIN twice, and confirm.
Keep your deletion PIN separate from your password. The PIN is an additional safeguard specifically for irreversible deletion actions. Do not write it down in the same place as your password. If you forget your PIN, a super_admin can reset it from the Team panel.

Super-Admin Docs link

The Super-Admin Docs (this documentation site) can be accessed from within the admin panel at any time. Click your name in the top-right profile dropdown and select "Super-Admin Docs". The docs open in a new tab.
