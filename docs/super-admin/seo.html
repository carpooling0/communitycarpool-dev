SEO & Search Management

This document covers all SEO assets on the platform — Open Graph tags, Google Search Console, robots.txt, sitemap.xml, and Schema.org — including what each does, how to update it, and how often.

On this page
Open Graph tags Google Search Console robots.txt sitemap.xml Google Analytics 4 Schema.org structured data New page checklist

Open Graph tags

Open Graph (OG) tags control how pages appear when shared on WhatsApp, LinkedIn, Twitter/X, and other platforms — the title, description, and image shown in the preview card.

Where they live

In the <head> of each HTML file:

<meta property="og:title" content="Community Carpool | Find Your Carpool Match" />
<meta property="og:description" content="Match with people on similar routes. Contact shared only after a match." />
<meta property="og:image" content="https://communitycarpool.org/og-image.jpg" />
<meta property="og:url" content="https://communitycarpool.org/" />
<meta name="twitter:card" content="summary_large_image" />

What each tag does

TagPurposeNotes
og:titleBold headline in the share cardKeep under 60 characters
og:descriptionSubtitle text below the headlineKeep under 160 characters
og:imageThumbnail image in the cardMust be 1200×630px, use og-image.jpg
og:urlCanonical URL for the pageAlways use full https:// URL
twitter:cardControls Twitter/X card typesummary_large_image for big image

Which pages have OG tags

OG tags are set on public-facing pages only. Private pages (matches, admin, unsubscribe, docs/admin, docs/super-admin) use <meta name="robots" content="noindex, nofollow"> instead and do not need OG tags.

The OG image (og-image.jpg — 1200×630px) is shared across all public pages. It is generated from the logo file Car_Pool_with slogan.png using a Python script.

How to verify OG tags

Use opengraph.xyz — paste any URL to see exactly how it will appear on WhatsApp, LinkedIn, Facebook, and Twitter/X.

  1. Go to opengraph.xyz
  2. Paste the full URL (e.g. https://communitycarpool.org)
  3. Check the preview for each platform
  4. Use the "Technical OG Tag Inspector" tab for the raw tag values

When to update OG tags

TriggerAction
New public landing page addedAdd og:title, og:description, og:url to that page. og:image stays as og-image.jpg.
Brand refresh or new taglineRegenerate og-image.jpg and update og:title / og:description on index.html
Homepage title or description changesUpdate og:title and og:description on index.html only

Google Search Console

Google Search Console (GSC) shows how the site performs in Google Search — which queries bring traffic, how pages rank, and any indexing issues.

Access

Go to search.google.com/search-console and sign in with the Community Carpool Google account. The property is verified via DNS TXT record on Cloudflare (domain-level verification — covers all subdomains).

Key reports to check monthly

ReportWhereWhat to look for
PerformanceSearch results → QueriesTop queries driving clicks; any dropping in position
CoverageIndexing → PagesAny pages with errors or excluded unexpectedly
SitemapsIndexing → SitemapsSitemap status — should show Success
Core Web VitalsExperience → Core Web VitalsAny pages flagged as Poor

How to request indexing for a new page

  1. In GSC, use the URL inspection tool (top search bar)
  2. Paste the full URL of the new page
  3. Click Request Indexing
  4. Google typically crawls within 24–72 hours

robots.txt

robots.txt tells search engine crawlers which pages they are allowed to index. It lives at the root of the repo: communitycarpool.org/robots.txt.

Current configuration

User-agent: *
Allow: /
Disallow: /admin.html
Disallow: /matches.html
Disallow: /unsubscribe.html
Disallow: /confirm-deletion.html
Disallow: /docs/admin/
Disallow: /docs/super-admin/

Sitemap: https://communitycarpool.org/sitemap.xml

Rules

  • Public pages (index, support, terms, docs user articles, landing pages) → Allow
  • Private pages (matches, admin, unsubscribe, confirm-deletion, admin docs, super-admin docs) → Disallow

When to update robots.txt

TriggerActionFrequency
New private page addedAdd Disallow: entry for that pathAs needed
New public page addedNo change needed — Allow: / covers all pages by default—
Routine reviewCheck that all private pages are still disallowedQuarterly

robots.txt does not need regular updates — only when the page structure changes (new private page added or a page changes from private to public).

sitemap.xml

sitemap.xml lists all public URLs on the site so Google can discover and crawl them efficiently. It lives at communitycarpool.org/sitemap.xml and is already submitted to Google Search Console.

When to update sitemap.xml

TriggerActionFrequency
New public page addedAdd a <url> entry with <loc>, <lastmod>, and <priority>Every time a new page goes live
Page removed or made privateRemove its <url> entryAs needed
Significant content updateUpdate the <lastmod> date for that URLAs needed
Routine reviewVerify all URLs return 200 and are still publicQuarterly

Priority guidance

PriorityPages
1.0Homepage (/)
0.8Key landing pages, docs index
0.6Individual docs articles, support, terms
0.4Secondary landing pages

How to add a new URL to sitemap.xml

<url>
  <loc>https://communitycarpool.org/dubai-carpool.html</loc>
  <lastmod>2026-04-01</lastmod>
  <changefreq>monthly</changefreq>
  <priority>0.8</priority>
</url>

After updating sitemap.xml

  1. Deploy to prod (git push origin main)
  2. Go to Google Search Console
  3. Navigate to Indexing → Sitemaps
  4. The sitemap is already submitted — GSC re-reads it automatically. If you want to force a re-read, delete and re-add: https://communitycarpool.org/sitemap.xml

Google Analytics 4

GA4 has been removed from the platform. Campaign attribution is handled natively via ref_code on the users table, populated from ?ref= URL parameters. Referral links and QR codes are managed in the admin dashboard.

If GA4 is reinstated in future, cookie consent gating must be implemented before the GA4 script fires (GDPR requirement).

Schema.org structured data

Schema.org JSON-LD blocks help Google understand page content and can unlock rich results (FAQ dropdowns, breadcrumbs, article metadata) in search.

What we have

Schema typePages
WebSite + Organizationindex.html
Article + BreadcrumbListAll 21 docs user articles
WebPagesupport.html, terms.html
WebPage + BreadcrumbListdocs/index.html

When to update schema

TriggerAction
New landing pageAdd WebPage JSON-LD from the landing page template
New docs articleAdd Article + BreadcrumbList JSON-LD
Organisation name or URL changesUpdate Organization schema on index.html

How to verify schema

Use Google's Rich Results Test — paste any URL to see detected schema and any errors.
