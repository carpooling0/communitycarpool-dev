User search & blacklist

Look up any registered user by email to view their account details, journeys, and match history. You can also add a user to the blacklist to prevent future submissions.

ℹ️
This panel is available to Admin and Super-Admin roles only. If you have a Support role and need access, contact a Super-Admin.

Searching for a user

Type an email address — full or partial — in the search bar and press Enter or click the search button.

  • If found: a user card appears below the search bar with the user's account details
  • If not found: a "No user found" message appears. The email may not be registered, may have been deleted, or you may have a typo — try with just the domain or username portion to narrow it down

The search checks the users table. Deleted accounts are not searchable here (they have been permanently removed from the database).

User card

The user card shows the following at a glance:

FieldMeaning
NameThe user's registered full name
EmailTheir email address
JoinedAccount creation date
Last seenThe last time they visited their matches page (matches.html). This is the most reliable activity signal.
Journey countTotal number of journeys ever submitted under this account (includes active, expired, and any that were paused during deletion pending)

Journeys

Click the Journeys section or expand arrow to see the full list of submissions for this user. Each journey shows:

  • From location — the user's origin address / area
  • To location — their destination
  • Created date — when the journey was submitted
  • Status — current status of the journey (see badges below)
  • Match count — how many matches have been found for this journey

Journey status badges

BadgeMeaning
ActiveJourney is live and being matched against other active journeys
ExpiredJourney has passed its expiry date — it is no longer being matched but its data is retained
Deletion pendingThe account has a pending deletion request — this journey will be deleted on the scheduled deletion date

Matches per journey

Click any journey row to expand and view its matches. This is useful when investigating a user complaint about missing matches or when reviewing a Report User ticket.

ColumnWhat it shows
Match IDInternal reference number for the match record
Other partyThe matched user's name, masked as "A**** B." — first initial, masked surname. This is by design: full contact details are only exchanged when both parties express mutual interest via the match email.
StrengthMatch quality as a percentage (e.g. 89%) based on route and timing overlap. Higher = better fit.
StatusCurrent status: new (match found, email not sent), notified (email sent to both parties), mutual_confirmed (both expressed interest), declined (one or both declined)
NotifiedTimestamp when the match notification email was sent to both parties
Interest AWhen this user expressed interest in the match (clicked the interest button in their match email)
Interest BWhen the other party expressed interest
💡
Why is the other party masked? Privacy is by design — contact details are only exchanged between users when both parties explicitly express interest via the match email. This protects users from unsolicited contact. Do not attempt to look up the other party separately to share their details with the first user.

This panel is read-only

The Users panel is for viewing information only. From this panel, you cannot:

  • Approve or decline matches on behalf of users
  • Change a journey's status or expiry date
  • Edit user details (name, email, preferences)
  • Send emails to users
  • Resend match notification emails

If any of these actions are needed, escalate to a Super-Admin — they can perform direct database updates for legitimate operational reasons.

Adding a user to the blacklist

The Add to Blacklist button (red ghost button) appears at the bottom of the user card after a successful search.

When to blacklist

  • The user has been confirmed (via a Report User ticket) to have harassed or threatened another user
  • The user is repeatedly submitting spam or abusive content in their journey entries
  • You have received a legal instruction to prevent this email from further activity on the platform

Do not blacklist based solely on an unverified complaint. Investigate first — check their journeys for any concerning content, review the Report User ticket in full, and if in doubt, check with a Super-Admin before proceeding.

How to blacklist

1

Find the user

Search for the user by email. Confirm you have the right account by checking their name and journey history.

2

Click Add to Blacklist

Click the red Add to Blacklist button at the bottom of the user card.

3

Confirm

A confirmation prompt will appear. Confirm the action to proceed. The user's email is added to the blacklist table immediately.

What blacklisting does

From the moment a user is blacklisted, any attempt to submit a new journey via the submit-journey edge function is rejected. They receive an error and cannot create a new submission.

What blacklisting does NOT do

  • It does not delete their existing data — current journeys and matches remain in the database
  • It does not revoke their access to their matches page — they can still view existing matches and respond to interest
  • It does not send the user any notification that they have been blacklisted
  • It does not prevent them from registering a new account with a different email address (that would require Super-Admin–level controls)

Removing someone from the blacklist

Removing an email from the blacklist currently requires Super-Admin intervention via a direct database update. If you believe a blacklisting was made in error or should be reversed, contact a Super-Admin with the reason.

⚠️
Blacklisting affects a real user's ability to participate in the platform. Confirm the reason is solid and documented before proceeding. Record your reasoning in the relevant support ticket notes so there is an audit trail.
