Campaign links

Referral and tracking links for partner organisations and campaigns. Each link carries a unique code that attributes new user signups to a specific campaign, organisation, or outreach effort.

ℹ️
This panel is available to Admin and Super-Admin roles only. If you have a Support role and need access, contact a Super-Admin.

What is a campaign link?

When a user visits the platform via a URL containing ?ref=CODE and submits a journey, that code is recorded against their submission in the user_attribution table. This lets you see which campaigns, partner organisations, or outreach efforts are driving sign-ups — and how many journeys each source has generated.

Example URL:

https://communitycarpool.org/?ref=ACME2026

When this link is shared with ACME Corp employees, every user who arrives via that URL and submits a journey will be attributed to the ACME2026 code.

The Campaign Links panel

The panel displays a table of all campaign links that have been created.

ColumnMeaning
CodeThe unique referral code (e.g. ACME2026). This is appended to the URL as ?ref=CODE.
LabelA human-readable name for the campaign — for internal reference (e.g. "ACME Corp — Office Poster March 2026")
OrganisationThe linked organisation, if any. Shown as the org name if an org record is attached; blank if no org is linked.
CreatedWhen the link was created and by whom
UsesTotal number of journey submissions that have used this code — the primary attribution metric
Last usedDate of the most recent submission attributed to this code
QRDownload button — click to download a QR code PNG that resolves to this campaign URL

Creating a new campaign link

1

Click + New Link

Click the + New Link button at the top of the Campaign Links panel. A creation form or modal will appear.

2

Fill in the details

  • Label (required) — a descriptive internal name. Use a format that will still make sense in 6 months, e.g. "ACME Corp — Office Poster March 2026" or "LinkedIn Campaign Q1 2026"
  • Organisation (optional) — select from the dropdown if this link is for a specific partner org. Leave blank for general campaigns.
  • Custom code (optional) — if you want a specific code (e.g. ACME2026), type it here. If left blank, a code is auto-generated (typically a short alphanumeric string). Codes must be unique — the system will warn you if the code is already in use.
3

Click Create

The new link appears in the table immediately with a Uses count of 0. The shareable URL is displayed in the row.

Sharing a campaign link

Two formats are available for each link:

Shareable URL

Format: https://communitycarpool.org/?ref=CODE

Share this with:

  • HR teams at partner organisations (for internal distribution)
  • Community managers and org contacts
  • In email newsletters, Slack/Teams messages, or social media posts
  • In email signatures for staff involved in promoting the carpool programme

QR code

Click the QR button in the table to download a PNG image of the QR code for that link. The QR code resolves to the campaign URL when scanned.

Use the QR code for:

  • Print flyers and posters in office buildings
  • Presentation slides
  • Physical notice boards in partner organisation facilities

The QR PNG is ready for print — you do not need to add any additional text to the image, though you may embed it in a designed flyer template.

Viewing attribution data

The Uses column shows the total number of journey submissions attributed to each code. This is the headline figure for reporting to a partner organisation — "X journeys submitted via your campaign link."

For more detailed attribution data (which specific users came through the link, or which journeys), a Super-Admin can run a SQL query directly:

SELECT u.name, u.email, ua.ref_code, ua.created_at
FROM user_attribution ua
JOIN users u ON ua.user_id = u.user_id
WHERE ua.ref_code = 'ACME2026'
ORDER BY ua.created_at DESC;

If you need this data for a report, request it from a Super-Admin.

Notes and limitations

  • Open links: Campaign links do not restrict who can use them — any user who visits the URL gets attributed to the code, regardless of whether they are associated with the named organisation. If an open link is shared publicly it can pick up unintended users.
  • First-touch attribution: each user can only be attributed to one referral code — the first one they used. If they visit the site via two different campaign links before submitting, only the first code is recorded.
  • No automated reporting: analytics are pull-based — check the panel or ask a Super-Admin for a query. There are no scheduled email summaries or automated reports to partners.
  • Link deletion: campaign links cannot currently be deleted from the panel — contact a Super-Admin if a link needs to be removed.
