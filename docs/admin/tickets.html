Support tickets

Users submit support tickets from support.html. This guide covers how to review, annotate, and close tickets — and when to notify the submitter by email.

What is a support ticket?

Users can submit support requests in two ways:

  • Report User — someone is behaving inappropriately, sending harassing messages, or posing a safety concern
  • Technical — the app is not working as expected: can't find matches, email not received, login issues, display errors

Deletion requests are handled through a separate automated flow and do not appear in the Tickets panel. If a user emails you asking about their account deletion, check the Deletions panel instead.

Finding tickets

At the top of the Tickets panel there are filter tabs showing a count for each category:

  • All — every ticket regardless of status
  • Open — tickets that still need attention
  • Closed — resolved tickets (kept for reference)

Use the search bar to find a specific ticket by email address (full or partial) or by ticket number — for example, type #42 to jump directly to ticket 42.

Handling a ticket — step by step

1

Open the ticket

Click the ticket row or the expand arrow to open the full details. You will see the user's message, the ticket type, submission time, and the note history (empty on new tickets).

2

Read the full message

Read the user's message carefully before taking any action. For Report User tickets, note what behaviour is being reported and whether it appears to involve a match interaction.

3

Add an internal note (optional but recommended)

Type your working notes in the Admin Notes textarea. This is for internal record-keeping — it is not visible to the user unless you check "Notify user" when closing. Examples of good notes:

  • "Reviewed — user cannot receive emails because their address has a typo. Advised them to re-register."
  • "Report User: reviewed the reported user's submissions — no matching records found. Cannot verify claim. Monitoring."
  • "No action required — duplicate of ticket #38."

Click Save Note to save the note. Notes are appended with a timestamp to the note history — you can add multiple notes over time.

4

Close the ticket

When you are ready to close, decide whether the user should receive an email notification:

  • Check "Notify user" if the user should be informed their ticket was reviewed. They will receive an email: "Your support ticket #X has been reviewed by our team." followed by your most recent admin note (if any). Use this for tickets where the user is expecting a response or where you have useful information for them.
  • Leave "Notify user" unchecked if the ticket should be closed silently (no email). Appropriate for spam submissions, duplicates, or internal-only issues where no user-facing response is needed.

Click Close Ticket. The status changes to Closed immediately.

⚠️
Closing a ticket cannot be undone from the admin panel. There is no re-open button. If a closed ticket genuinely needs to be reopened (for example, the user replies with new information), a Super-Admin can update the status directly in the database.

Note history

  • Every saved note is appended to the note history with the timestamp and the name of the admin who wrote it
  • Notes are internal — they are only visible to admin panel users, not to the ticket submitter
  • The note included in the close notification email (if "Notify user" is checked) is your most recent note at the time of closing
  • Notes cannot be edited after saving — they are append-only. If you need to correct a note, add a new one with the correction.

Best practices

  • Always add a note before closing, even if brief — for example: "No action required — spam submission" or "Confirmed issue, advised user to clear cache and retry"
  • For Report User tickets: document what you reviewed and what action (if any) was taken, even if the result is "unverified, monitoring". If you look up the reported user in the Users panel, note that in the ticket.
  • For Technical tickets: if the issue appears to be an edge function error, missing data, or a system-level problem (rather than a user error), escalate to Super-Admin before closing the ticket
  • Do not leave Open tickets unattended for more than 3 business days — check the Tickets panel at least weekly

What you cannot do from this panel

  • Reopen a closed ticket — Super-Admin can update the DB directly if truly needed
  • Edit a submitted note — notes are append-only
  • See the submitter's full match data from here — use the Users panel to look up their account and match history
  • Take action on the reported user — if a Report User ticket requires blacklisting, switch to the Users panel and search for that user
